# Envlink API Documentation

Welcome to the Envlink SaaS URL Shortener API documentation. Our API provides a robust and efficient solution for creating, managing, and tracking shortened URLs. It is designed to be both powerful and easy to use, catering to a wide range of applications, from simple link shortening to complex analytics and subscription management.

This documentation will guide you through the various features and endpoints available in our API.

## API Reference

The following is a list of available API documentation. For full API documentation, visit [docs.envlink.one](https://docs.envlink.one).

### Core APIs

- **Account API**: Manage user authentication, profiles, and security. ([Details](pages/account.mdx))
- **Analytics API**: Get detailed statistics and tracking for your short URLs. ([Details](pages/analytics.mdx))
- **Authentication API**: Handle user registration, login, and token management. ([Details](pages/authentication.mdx))
- **URLs API**: Create, retrieve, update, and delete short URLs. ([Details](pages/urls.mdx))
- **User API**: Manage user profile information. ([Details](pages/user.mdx))

### Billing & Subscriptions

- **Billing Address API**: Manage user billing addresses. ([Details](pages/billing-address.mdx))
- **Payment Methods API**: Manage user payment methods. ([Details](pages/payment-methods.mdx))
- **Subscriptions API**: Manage user subscriptions, plans, and billing cycles. ([Details](pages/subscriptions.mdx))
- **Transactions API**: Retrieve user transaction history. ([Details](pages/transactions.mdx))

### Public APIs

- **Public Payment Methods API**: Allow unauthenticated users to manage payment methods. ([Details](pages/public-payment-methods.mdx))
- **Public Subscriptions Plans API**: Retrieve information about available subscription plans. ([Details](pages/public-subscriptions-plans.mdx))
- **Public URLs API**: Create and manage short URLs without authentication. ([Details](pages/public-urls.mdx))

### Utility APIs

- **Health API**: Monitor the health status of the application. ([Details](pages/health.mdx))
- **OAuth API**: Authenticate users via third-party providers like Google. ([Details](pages/oauth.mdx))
- **Sessions API**: Manage and revoke user sessions for enhanced security. ([Details](pages/sessions.mdx))
- **Webhooks API**: Handle incoming webhook notifications from services like Xendit. ([Details](pages/webhooks.mdx))
